# animista (full)

> Full source of the **animista** skill (catalog: [[animista]]). Pure hand-written CSS, no JS/build.

**description:** Generate ready-to-paste CSS keyframe animations in the style of animista.net —
entrance/exit, attention, text, background (scale, slide, rotate, flip, slit, swing, roll,
bounce, fade, puff, flicker, tracking, focus, vibrate, shake, jello, heartbeat, pulsate…).
Each ships the recognizable Animista header comment, a usage class, the full `@keyframes`, and
tunable params (duration, delay, timing, iteration, direction, fill). Use whenever the user
wants a CSS animation by name/feel ("slide in from left", "bounce-in", "blur into focus",
"vibrate/shake", "animista-style"), a catalog, or a playground page. **Not for:** React/Framer
Motion (→ framer-motion), GSAP/Three.js scroll, or full page/section design (→ precision-bento).

## Output format (always)
```css
/* ----------------------------------------------
 * Generated by Animista on 2024-1-1 0:0:0
 * Licensed under FreeBSD License.
 * See http://animista.net/license for more info.
 * w: http://animista.net, t: @cssanimista
 * ---------------------------------------------- */
.slide-in-left {
  -webkit-animation: slide-in-left 0.5s cubic-bezier(0.250,0.460,0.450,0.940) both;
          animation: slide-in-left 0.5s cubic-bezier(0.250,0.460,0.450,0.940) both;
}
@keyframes slide-in-left {
  0%   { transform: translateX(-1000px); opacity: 0; }
  100% { transform: translateX(0); opacity: 1; }
}
```
- **Header comment** always present (the signature). **`-webkit-` prefixes** by default (drop on "modern/clean"). **`both`** fill mode for one-shots. **Class name = animation name.** One animation per block.

## Process
1. Map request → named animation (feel→name table / `references/animations.md`).
2. Pick easing from Animista's palette (don't invent curves).
3. Set params (per-family defaults + user overrides).
4. Emit block with `@keyframes` **verbatim** (don't paraphrase percentages).
5. State the trigger (load / `:hover` / scroll via IntersectionObserver / `infinite`).

## Easing palette (named cubic-beziers)
ease-out `0.250,0.460,0.450,0.940` (entrances) · ease-in `0.550,0.085,0.680,0.530` (exits) ·
ease-in-out `0.455,0.030,0.515,0.955` · out-cubic `0.215,0.610,0.355,1.000` ·
out-quint `0.230,1.000,0.320,1.000` · scale `0.390,0.575,0.565,1.000` ·
overshoot `0.175,0.885,0.320,1.275` · anticipate+overshoot `0.680,-0.550,0.265,1.550` ·
puff `0.470,0.000,0.745,0.715`. (Bounce/heartbeat/flicker carry per-keyframe timing — don't override.)

## Default params by family
scale/fade/slide/rotate `0.4–0.6s` ease-out (`both`) · flip/slit/swing `0.4–0.5s` (needs parent
`perspective`) · bounce-in `1.1s` (per-keyframe, no easing) · roll/puff `0.6–0.7s` ·
tracking/focus `0.7–0.8s` cubic · flicker `2s` linear · vibrate/shake `0.3–0.8s` (+`infinite`) ·
heartbeat/pulsate/ping/blink `1.5s` `infinite both`.

## Triggering
- Load: add class. Hover: `.el:hover`. Scroll: IntersectionObserver 6-liner. Loop: add `infinite`.

## Ground rules
- ALWAYS include the header comment + verbatim `@keyframes`; class name = animation name; `both` for one-shots.
- Prefer a palette curve over `ease`/`linear` guesses. `-webkit-` by default. Flip/slit/swing → remind to set `perspective`.
- NEVER use JS libs/GSAP/Framer Motion here (hand off to `framer-motion`). Respect `prefers-reduced-motion`.
- Reference: `references/animations.md` (full catalog + exact keyframes), `references/playground.md`, `assets/playground.html` (no-build playground).
