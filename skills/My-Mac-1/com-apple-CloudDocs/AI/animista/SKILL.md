\# Animista  
  
A reusable \*\*CSS animation library\*\* in the spirit of \[animista.net\](https://animista.net) —  
on-demand, on-scroll, hover, and continuous keyframe animations you copy straight into a  
stylesheet. The value is \*\*fidelity\*\*: animations that match Animista's actual output, named  
the same way, carrying the same signature header, easing curves, and parameters — so the result  
looks and feels like it came from the real tool.  
  
\#\# What this skill does  
  
Given a request for an animation (by name, by feel, or "give me X-style"), emit a complete,  
self-contained CSS block:  
  
1\. The \*\*Animista header comment\*\* (the fingerprint — always include it).  
2\. A \*\*usage rule\*\* on a target class/selector with the full \`animation\` shorthand.  
3\. The \*\*\`@keyframes\`\*\* definition.  
  
It can also scaffold a \*\*live playground page\*\* (an animista.net clone) — a grid of animation  
buttons, a preview stage, parameter controls, and a generated-CSS panel with copy — from  
\[\`assets/playground.html\`\](assets/playground.html).  
  
\#\# Output format (always use this shape)  
  
\`\`\`css  
/\* ----------------------------------------------  
 \* Generated by Animista on 2024-1-1 0:0:0  
 \* Licensed under FreeBSD License.  
 \* See http://animista.net/license for more info.  
 \* w: http://animista.net, t: @cssanimista  
 \* ---------------------------------------------- \*/  
  
.slide-in-left {  
 -webkit-animation: slide-in-left 0.5s cubic-bezier(0.250, 0.460, 0.450, 0.940) both;  
 animation: slide-in-left 0.5s cubic-bezier(0.250, 0.460, 0.450, 0.940) both;  
}  
  
@-webkit-keyframes slide-in-left {  
 0% { -webkit-transform: translateX(-1000px); transform: translateX(-1000px); opacity: 0; }  
 100% { -webkit-transform: translateX(0); transform: translateX(0); opacity: 1; }  
}  
@keyframes slide-in-left {  
 0% { -webkit-transform: translateX(-1000px); transform: translateX(-1000px); opacity: 0; }  
 100% { -webkit-transform: translateX(0); transform: translateX(0); opacity: 1; }  
}  
\`\`\`  
  
\- \*\*Header comment\*\*: always present. It is the recognizable Animista signature.  
\- \*\*\`-webkit-\` prefixes\*\*: include them by default (this is how Animista emits code). If the  
 user says "no prefixes" or "modern only", drop the \`-webkit-\` lines and emit clean CSS.  
\- \*\*\`both\` fill mode\*\*: the default so the element holds its start frame before and end frame  
 after — entrances should not flash visible→hidden→animate.  
\- \*\*Class name = animation name\*\*: keep them identical (\`.slide-in-left { animation: slide-in-left ... }\`).  
\- One animation per block. For multiple, stack blocks under one header.  
  
\#\# Process  
  
\- \[ \] \*\*Map the request to a named animation.\*\* Use the feel→name table below, or the full  
 catalog in \[references/animations.md\](references/animations.md). If ambiguous (e.g. just  
 "bounce"), pick the most common variant and name it ("I used \`bounce-in-top\` — say the  
 word for left/right/fwd/bck").  
\- \[ \] \*\*Pick the easing\*\* from the Animista palette (below) that matches the family — do not  
 invent curves. Entrances → smooth ease-out; exits → smooth ease-in; playful → overshoot.  
\- \[ \] \*\*Set parameters.\*\* Default duration per family (below). Apply any user overrides for  
 duration, delay, iteration-count, direction, fill-mode.  
\- \[ \] \*\*Emit the block\*\* in the exact output format. Include the \`@keyframes\` verbatim from the  
 catalog — do not paraphrase the percentages.  
\- \[ \] \*\*State how to trigger it\*\* if relevant: on load (just add the class), on hover  
 (\`.el:hover\`), on scroll (toggle the class via IntersectionObserver — give the 6-line  
 snippet), or looping (\`infinite\`).

\#\# Easing palette (Animista's named curves — use these, don't invent)  
  
| Feel | \`cubic-bezier(...)\` |  
|---|---|  
| ease-out (smooth landing) — default for entrances | \`0.250, 0.460, 0.450, 0.940\` |  
| ease-in (smooth pickup) — default for exits | \`0.550, 0.085, 0.680, 0.530\` |  
| ease-in-out (smooth both) | \`0.455, 0.030, 0.515, 0.955\` |  
| ease-out cubic (a touch sharper) | \`0.215, 0.610, 0.355, 1.000\` |  
| ease-out quint (long glide) | \`0.230, 1.000, 0.320, 1.000\` |  
| scale curve (used by scale-\*) | \`0.390, 0.575, 0.565, 1.000\` |  
| overshoot at end (back/playful) | \`0.175, 0.885, 0.320, 1.275\` |  
| anticipate + overshoot (strong) | \`0.680, -0.550, 0.265, 1.550\` |  
| puff curve | \`0.470, 0.000, 0.745, 0.715\` |  
  
Bounce, heartbeat, and flicker animations carry per-keyframe \`animation-timing-function\` and  
run on \`linear\`/\`ease-in-out\` — do not override their shorthand easing.  
  
\#\# Default parameters by family  
  
| Family | Duration | Easing | Notes |  
|---|---|---|---|  
| scale, fade, slide, rotate | \`0.4s\`–\`0.6s\` | ease-out / scale curve | \`both\` |  
| flip, slit, swing | \`0.4s\`–\`0.5s\` | ease-out / overshoot | needs \`perspective\` on parent for depth |  
| bounce-in | \`1.1s\` | (per-keyframe) | longer; do not add easing |  
| roll, puff | \`0.6s\`–\`0.7s\` | ease-out / puff curve | |  
| tracking, focus | \`0.7s\`–\`0.8s\` | cubic | text effects |  
| flicker | \`2s\` | linear | \`both\` |  
| vibrate, shake | \`0.3s\`–\`0.8s\` | linear / ease-in-out | add \`infinite\` for continuous |  
| heartbeat, pulsate, ping, blink | \`1.5s\` | ease-in-out / linear | \`infinite both\` |  
  
\#\# Triggering (state it when relevant)  
  
\- \*\*On load:\*\* add the class to the element — it runs once with \`both\`.  
\- \*\*On hover:\*\* put the rule on \`.el:hover\` (or use a continuous one like \`vibrate-1\`).  
\- \*\*On scroll into view:\*\* ship this 6-liner so the animation fires when visible:  
 \`\`\`js  
 const io = new IntersectionObserver((es) =\> es.forEach(e =\> {  
 if (e.isIntersecting) { e.target.classList.add('slide-in-left'); io.unobserve(e.target); }  
 }), { threshold: 0.2 });  
 document.querySelectorAll('.reveal').forEach(el =\> io.observe(el));  
 \`\`\`  
\- \*\*Looping:\*\* add \`infinite\` to the shorthand (\`vibrate-1 0.3s linear infinite both\`).

\#\# Ground rules  
  
\- ALWAYS include the Animista header comment block — it is the signature.  
\- ALWAYS copy \`@keyframes\` percentages and transforms verbatim from the catalog. Do NOT  
 paraphrase or "clean up" multi-stop bounce/flicker/heartbeat frames — the exact stops are  
 what make them read correctly.  
\- ALWAYS use \`both\` fill mode for one-shot entrances/exits unless told otherwise.  
\- ALWAYS name the class identically to the animation name.  
\- PREFER an Animista easing curve from the palette over \`ease\`/\`linear\` guesses (except where a  
 family mandates per-keyframe timing).  
\- INCLUDE \`-webkit-\` prefixes by default; drop them only when the user asks for modern/clean CSS.  
\- For flip/slit/swing 3D animations, REMIND the user to set \`perspective\` (e.g. \`600px\`) on the  
 parent or \`transform: perspective(...)\` so the rotation reads as depth.  
\- NEVER reach for JS animation libraries, GSAP, or Framer Motion here — this skill is pure CSS.  
 If the user wants React motion, hand off to the \`framer-motion\` skill.  
\- RESPECT reduced motion: when building a full page, wrap continuous/large motion in  
 \`@media (prefers-reduced-motion: no-preference) { ... }\` or null it under \`reduce\`.
