## Why this manifest exists
Drive files can only be pulled through a tool that returns each file as base64
**into the model's context**. That imposes a hard ceiling: large binaries
(multi-MB PDFs, `.pages`, big images) cannot be routed, and reproducing large
base64 blobs verbatim is unreliable. So small authored text files were imported
directly; everything else is indexed below with a Drive link so it can be
pulled manually or by a direct sync tool (e.g. `rclone`).
